loadTotals (   $columns = array())

Load the totals for the current found set using MySQL's aggregate operators. This method will always query the database directly (no caching).

Parameters
array$columns
Returns
type
Exceptions
Exception<p>Thismethod was developed to make it easier to create "totals" rows in the list view.

Example

$resultSet = Dataface_Application::getInstance()->getResultSet();
$results = $resultSet->loadTotals(array(
'age#avg',
'age#std',
'income#sum'
));
print_r($results);

Output:

array(
'age_AVG' => 74.4,
'age_STD' => 13.2,
'income_SUM' => 100567
);
Parameters
array$columnsA list of strings of the form {fieldname}#{operator} where where {fieldname} is the name of a field and {operator} is the name of an operator to apply to the field.

Available Operators

OperatorDescription
sumThe sum of field values for result set.
avgThe average value of field in result set.
maxThe max value of field in result set.
minThe minimum value of field result set.
bit_andResult of bitwise AND on field for all found records.
bit_orResult of bitwise OR on field for all found records.
bit_xorResult of bitwise XOR on field for all found records.
group_concatThe concatenation of all rows in found set for field. Joined with a comma.
stdPopulation Standard deviation of field values for result set.
stddev_sampSample Standard deviation of field values for result set.
stddev_popSame as stddev
var_popPopulation standard variance for field values for result set.
var_sampSample standard deviation for field values for result set.
varianceSame as var_pop
Returns
Associative array where the keys have the form {fieldname}_{operator|uppercase} and the values are the corresponding result of the operator on that field. E.g. If your input was array('field1::sum', 'field2::avg') the output woult be array('field1_SUM' => {some number}, 'field2_AVG' => {some_number})
Since
2.0.4
